# セキュリティポリシー

KeruLabs のセキュリティに関心をお寄せいただきありがとうございます。このドキュメントでは、脆弱性の報告方法とプロジェクトのセキュリティ設計について説明します。

## 脆弱性の報告

### 報告方法

セキュリティ上の脆弱性を発見した場合は、**公開 Issue として報告しないでください**。代わりに、以下の方法でご連絡ください:

GitHub の **[Private vulnerability reporting](https://github.com/takataka6/kerulabs/security/advisories/new)** 機能を通じて、非公開のセキュリティアドバイザリを作成してください。

### 報告に含めるべき情報

- 脆弱性の種類（XSS、インジェクション、権限昇格など）
- 影響を受けるファイルやコンポーネントのパス
- 再現手順（ステップバイステップ）
- 影響範囲の評価
- 可能であれば修正案

### 対応フロー

1. **確認**: 報告受領後、3 営業日以内に確認のご連絡をします
2. **調査**: 脆弱性の影響範囲を調査し、修正計画を策定します
3. **修正**: パッチを作成し、テストを実施します
4. **公開**: 修正リリース後、適切な期間を置いてアドバイザリを公開します

### 対象バージョン

| バージョン | サポート状況   |
| ---------- | -------------- |
| 0.4.x      | サポート対象   |
| 0.3.x 以前 | サポート対象外 |

## Electron のセキュリティ設定

KeruLabs は Electron を使用したデスクトップアプリケーションとしても動作します。以下のセキュリティ設定を適用しています。

### BrowserWindow 設定

```typescript
webPreferences: {
  preload: path.join(__dirname, "preload.js"),
  nodeIntegration: false,     // Node.js API をレンダラープロセスで無効化
  contextIsolation: true,     // プリロードスクリプトとWebコンテンツを分離
}
```

- **`nodeIntegration: false`**: レンダラープロセスから Node.js の API（`fs`, `child_process` 等）に直接アクセスすることを禁止しています。これにより、XSS 攻撃が発生した場合でもファイルシステムへのアクセスを防ぎます。
- **`contextIsolation: true`**: プリロードスクリプトのコンテキストとWeb ページのコンテキストを分離しています。Web ページ側から Electron の内部 API やプリロードスクリプトのグローバル変数にアクセスすることはできません。

### IPC 通信

プリロードスクリプトを通じた**チャネルホワイトリスト方式**を採用しています:

- レンダラープロセスから使用可能な IPC チャネルは、プリロードスクリプトで明示的に公開されたもののみです
- 任意のチャネルへの送信はできません

### Content Security Policy (CSP)

Electron ではレスポンスヘッダーとして、Web 版では Vite dev / preview サーバーのレスポンスヘッダーとして CSP を適用しています。静的 Web bundle を任意のサーバーで配信する場合は、配信サーバー側で同等の CSP を設定してください。

| ディレクティブ    | 値                                                                           | 目的                                                                               |
| ----------------- | ---------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `default-src`     | `'self'`                                                                     | デフォルトで自身のオリジンのみ許可                                                 |
| `script-src`      | `'self' blob: 'unsafe-eval'`                                                 | Three.js シェーダー・Mermaid・drei worker 対応                                     |
| `style-src`       | `'self' 'unsafe-inline'`                                                     | インラインスタイル許可                                                             |
| `img-src`         | `'self' data: blob:`                                                         | Data URL と Blob URL を許可                                                        |
| `font-src`        | `'self' data:`                                                               | ローカルフォントと Data URL を許可                                                 |
| `connect-src`     | `'self'`（本番）/ `'self' http://localhost:5173 ws://localhost:5173`（開発） | 開発時の HMR と同一オリジン通信を許可                                              |
| `worker-src`      | `'self' blob:`                                                               | Three/drei の Web Worker 用 Blob URL を許可                                        |
| `object-src`      | `'none'`                                                                     | プラグイン（Flash 等）を完全禁止                                                   |
| `base-uri`        | `'self'`                                                                     | base タグの制限                                                                    |
| `form-action`     | `'self'`                                                                     | フォーム送信先の制限                                                               |
| `frame-ancestors` | `'none'`                                                                     | iframe への埋め込み禁止。Electron と Vite preview / dev のレスポンスヘッダーで適用 |

### セキュリティヘッダー

| ヘッダー                 | 値                                | 目的                            |
| ------------------------ | --------------------------------- | ------------------------------- |
| `X-Content-Type-Options` | `nosniff`                         | MIME タイプスニッフィングの防止 |
| `X-Frame-Options`        | `DENY`                            | クリックジャッキングの防止      |
| `Referrer-Policy`        | `strict-origin-when-cross-origin` | リファラー情報の制限            |

注記:

- `script-src 'unsafe-eval'` は Three.js / Mermaid との互換性のために許可しています
- Vite dev server では React Refresh 互換性のため `script-src 'unsafe-inline'` を追加で許可しています
- `connect-src` の `localhost:5173` / `127.0.0.1:5173` は開発環境用です

## IndexedDB のクライアントサイドデータに関する注意事項

KeruLabs はデータの永続化に IndexedDB を使用しています。以下の点にご注意ください。

### データの保存場所

- すべてのデータ（チーム情報、戦術データ、フォーメーションなど）はクライアントサイドの IndexedDB に保存されます
- サーバーへのデータ送信は行いません
- データはユーザーのブラウザ（またはElectron アプリ）のローカルストレージに格納されます

### セキュリティ上の考慮事項

1. **暗号化されていないデータ**: IndexedDB のデータはブラウザ内に平文で保存されます。機密性の高い情報（パスワード、個人情報など）の保存には適していません。KeruLabs は戦術データ、チーム情報、用語集、チームマニュアルなどを保存するため、機微情報の入力は避けてください。

2. **同一オリジンポリシー**: IndexedDB は同一オリジンポリシーにより保護されています。他のドメインからデータにアクセスすることはできません。

3. **データの永続性**: ブラウザのストレージクリアやキャッシュ削除によりデータが失われる可能性があります。重要なデータはファイルエクスポート機能（JSON）を使用してバックアップしてください。

4. **XSS 対策**: ユーザー入力データを表示する際は DOMPurify によるサニタイゼーションを実施しています。IndexedDB から読み出したデータであっても、HTML として挿入する前に必ずサニタイズされます。

5. **ブラウザ拡張機能**: ブラウザ拡張機能は IndexedDB のデータにアクセスできる可能性があります。信頼できない拡張機能のインストールにはご注意ください。

### 開発者向けガイドライン

- IndexedDB に保存するデータには Zod スキーマによるバリデーションを適用してください
- ユーザー入力を含むデータの表示時は必ず DOMPurify でサニタイズしてください
- 新しいデータストアを追加する場合は、保存するデータに機密情報が含まれないことを確認してください

## 依存パッケージのセキュリティ

- 定期的に `pnpm audit` を実行し、既知の脆弱性を確認してください
- 依存パッケージの更新時は CHANGELOG を確認し、セキュリティ修正が含まれているか確認してください

セキュリティ脆弱性を責任ある方法で報告してくださった方には、CHANGELOG およびリリースノートで謝辞を掲載させていただきます。
