# ADR-0001: Clean Architecture の採用

## ステータス

採用済み

## コンテキスト

サッカー戦術教育アプリとして、ドメインロジック（戦術・フォーメーション・チーム管理）が複雑であり、UIフレームワークやデータベース実装に依存しない設計が求められた。

## 決定

Clean Architecture（ヘキサゴナルアーキテクチャ）を採用し、4層に分離する。

```
src/
├── domain/          # エンティティ、値オブジェクト、ドメインイベント
├── application/     # ユースケース（Interactor）、ポートインターフェース
├── infrastructure/  # IndexedDB リポジトリ、外部サービス実装
└── presentation/    # React コンポーネント、カスタムフック
```

**依存の方向**: presentation → application → domain ← infrastructure

## 結果

### メリット
- ドメイン層が外部フレームワークに依存せず、純粋な TypeScript で記述可能
- テスタビリティ向上（モックなしでドメインロジックをテスト可能）
- UI フレームワーク（React）を将来変更しても、ドメイン・アプリケーション層は影響を受けない
- ポートインターフェースにより、IndexedDB 以外のストレージへの移行が容易

### トレードオフ
- ボイラープレートが増加（ポート定義、マッピング関数）
- 小規模な機能追加でも複数レイヤーの変更が必要になる場合がある
